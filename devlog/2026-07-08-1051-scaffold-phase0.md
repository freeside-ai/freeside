# Phase 0 scaffolding

First session. Structure, conventions, and documents only; no code, per the
bootstrap's explicit out-of-scope list (no Go module, Swift package, OpenAPI
content, pipeline schema, or CI).

## Decisions

- **Empty root commit renamed `Initialize` and force-pushed to `main`.** The
  repo had no commits, so an empty root was created to let all real content
  land via PR (bootstrap §1). Renamed from "Root commit" at user request;
  amending an empty root needs `--allow-empty`. Force-pushing `main` is
  normally forbidden (AGENTS.md Commits), but this rewrote the empty root
  before any PR history existed and is a one-time pre-history fixup.
- **Component READMEs state purpose + toolchain + scope boundary only**
  (bootstrap §2). The scope-boundary line is load-bearing: it encodes where
  generated code lives (`api/` consumers, never `api/`), which directory is
  gated (`pipelines/`), and the anticipated `images/` repo split, so the
  monorepo scope discipline has a per-directory anchor.
- **AGENTS.md project-specific sections written, not left as placeholders.**
  The skill defaults to placeholders for a pre-code repo, but the bootstrap
  specified Monorepo scope discipline, Document gating, a placeholder
  build/test/run table, and a docs-coherence `done` block, so those went in
  now. Chose to fill them because they govern this very session's work
  (scope declaration, plan gating), not future code.
- **Repo settings aligned** (user-approved): auto-delete merged branches on,
  squash/rebase merge off, merge message = PR title only. Makes the
  `--first-parent` / merge-commit conventions real rather than manual
  fallbacks.
- **License deferred** (user choice). Philosophy classifies the repo as a
  network service (dominant component is `freesided`, an HTTP+WebSocket
  daemon) → AGPL-3.0 if licensed. But the repo is private, has no code, and
  open-sourcing is an explicit Phase 4 option (plan §8, decision #15).
  Licensing a private empty repo would commit a deliberately-held option, so
  left private and unlicensed. See ADR-candidate below.

## Fixed

- **api/ README status corrected.** First draft said "empty until Phase 1"
  while the same file called the OpenAPI skeleton a Phase 0 design artifact, a
  self-contradiction Codex flagged (P2). Aligned to the plan (§8): the
  skeleton is the first content to land in `api/`, drafted in Phase 0 ahead of
  the daemon's Phase 1 code. Initial sweep covered the component READMEs only;
  Codex's re-review then caught the standing AGENTS.md intro, which still
  blanket-declared every dir "empty pre-Phase-1" (wrong for `api/` at Phase 0
  and `app/` at Phase 2) and so would read as a scope-violation trap for the
  Phase 0 `api/` work. Widened the fix: the intro no longer asserts a blanket
  phase, carves out `api/`'s Phase 0 skeleton as in-scope, and defers
  per-component phases to each README and the plan. The `api/`=Phase-1 claim
  recurred across three surfaces — the README, the AGENTS.md intro prose, and
  this entry's own Deferred queue — each caught a round later. Lesson: a
  phase-claim sweep must cover every surface an agent reads at session start
  (component READMEs, AGENTS.md, and the devlog's open queues), not just the
  first one cited; a `git grep` of the whole class closes it in one push.
- **PR template gained a required `## Scope` section** (Codex, P2). The Monorepo
  scope discipline and the "Scope declared" done-check require every PR to name
  its touched component dirs, but the scaffolded template agents copy had no
  such field, so the checklist could pass while missing a done item. Class:
  a project-specific required convention not reflected in the scaffold.
- **Dropped the "free as in bird" brand line from the root README** (user
  request). Kept in the plan (§11) as the naming record; the landing page
  doesn't carry it pre-release.

## To promote

  This repo's devlog protocol (`devlog/README.md`: `YYYY-MM-DD-HHMM-slug.md`
  filenames, optional structure, no required front-matter) and the Freeside
  devlog *output contract* in plan §7 (one file per run in `devlogs/`,
  required front-matter: run ID / stage / date / linked issue-PR) differ in
  directory name, filename scheme, and front-matter. They must be reconciled
  before pipeline stages start emitting entries, or stage output won't match
  the protocol agents read. Expected resolution: extend this protocol with
  optional run front-matter (run ID, stage, linked issue/PR) so a
  stage-emitted entry is a valid manual entry with extra fields; this entry
  already carries such front-matter as a trial. Needs a human/Phase-1
  decision; escalate to a tracker issue when the issue tracker is used.
  -> resolved in 2026-07-13-1026-plan-v4-reconcile.md via plan v4 §9's
  cadence split: repo protocol unchanged for human sessions, autonomous
