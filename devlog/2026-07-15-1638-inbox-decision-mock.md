# Saddle inbox and decision detail against the mock (#71)

## Decisions

- **Grow the mock into a stateful contract mock inside FreesideAPI, not a scripted stub or a separate target.** The MockServer actor implements the contract's semantics over an in-memory item table (idempotent submission by `command_id`, optimistic-concurrency rejection carrying the current snapshot as `replacement_item`), and a thin transport routes generated-client calls through real JSON so every test crosses the full generated encode/decode pipeline. Scenario control is seeding plus two hooks (`advance` to force staleness, `beforeRespond` to hold open or fail a response), not per-test canned bodies. Rejected: operation-keyed canned responses, because the unit's acceptance criteria are interaction sequences (submit, conflict, retry) that canned bodies cannot express without scripting a state machine anyway; a separate mocks package, because it would still depend on FreesideAPI for generated types and the sync/cache unit extends this same server.
- **Encode fixtures as Swift builders over generated types, not JSON resources.** Generated source is not committed (bootstrap decision), so builders make schema drift a compile error where JSON fixtures would fail at decode time; the §4 per-type action table lives once in `AttentionFixtures.phase1ActionSets` and an independent transcription in `FixtureTests` pins it, the golden-test discipline adapted to a platform without committed generated shapes.
- **`blocked` ships a placeholder action set (`acknowledge / snooze / stop`).** Plan §4 defines action sets for nine of the ten Phase 1 types; `blocked` (§5.12) has none. #23 resolved mid-review (#97): signet's authoritative policy pins `blocked` read-only with no actions, but the API contract still requires a non-empty `requested_decision`, and #96 owns that relaxation. The fixture stays schema-valid with the placeholder until #96 lands, then switches to an empty set; the other nine sets match the merged policy exactly.
- **Read-your-write renders only from the received CommandResult; canonical state comes from a post-commit refetch, never a local resolve.** "Applied" state is set exclusively in the 200 path and a pending or transport-failed submission never renders as applied. The 200 carries only the record and revision, and plan §4 makes several actions non-resolving (`open_pr` navigates; `acknowledge` and `mark_seen` mean seen, never resolved), so the model refetches the item after commit and the mock mirrors the signet boundary's actionOutcome table decision-for-decision (concluding / record-only / pending-rejected; `daemon/internal/signet/service.go`), a provisional client mirror pending a queryable contract representation (#22). Closed items share the API's 409 replacement-snapshot shape with staleness per the recorded #65 decision (2026-07-15-1655). Rejected (on Codex's P1 in review): resolving locally on every 200, which hid still-open items behind non-terminal actions.
- **Every submission claims a store-owned per-item pending slot before its first request leaves the model, and the slot blocks every other command for the item.** An in-flight command can commit after any refetch, and a transport loss or 5xx proves nothing about commitment, so the original ClientCommand is preserved in `InboxStore` (surviving card navigation and model recreation, including mid-flight recreation) until a definitive outcome — 200, 409, or an authoritative 4xx — releases it; an ambiguous outcome resolves only by a verbatim resend that returns the recorded result or an authoritative rejection (§5.14 test 4). Rejected (owner review, two rounds): model-scoped retry state, which navigation silently dropped; heuristic settling that let a new command mint while an earlier outcome was still ambiguous; and failure-time registration, which left a suspension window where a recreated card could double-submit.

Revisit when #96 lands (drop the `blocked` placeholder for an empty action set) and when #72 introduces real cache semantics (the in-memory store and its pending-command ledger are its raw material).
