  terminality rule; the issue's own Contract note anticipated the
  relabel. It heads the chain because #65 builds directly on it.
- **The sixteen §5.14 tests are distributed as unit acceptance, not a
  separate test unit**: 1/2/4 → #65, 3/8/11 → #66, 13–16 → #67,
  5/6/7/10/12 → #68, 9 → #69; the client halves ride #72. Rejected: a

  workspace mount in the fresh exporter VM; output, blobs plus the
  normalized manifest on the exporter rootfs, collected via container
  export of the stopped exporter. Rejected: leaving the seam to be
  negotiated mid-lane (the exact failure mode the filing-time rule
  exists to prevent), and rejected: moving the helper into ward's scope
  (export normalization is gauntlet territory; ward owns lifecycle and
  image pinning only).
- **Ward decomposition follows the spike verbatim**: backend lifecycle

## Owner decisions recorded (binding on ward units)

- **The same-VM fallback class is refuted, not merely weaker, and must
  not be implemented.** The spike disproved it by execution on Apple
  container 1.1.0: no host hot-detach exists, and guest unmount even
  with CAP_SYS_ADMIN leaves the credential block device attached and
  remountable. §5.7's "honest fallback" language predates this
  evidence for this runtime; #79 queues the plan correction. #77's
  third negative probe exists so no future session reopens the class.
- **Workspace-copy export cost is accepted for the first repository.**
  1.1.0 has no direct named-volume export, so the helper copies the
  read-only workspace into the exporter rootfs. Revisit when: a public
  named-volume export/snapshot API lands in Apple container, a
  Freeside-owned Containerization integration with its own conformance
  contract is undertaken, or first-repository export latency becomes a
  measured burden. Never via the spike's private block-image
  manipulation.
